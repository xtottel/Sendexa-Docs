# SMPP Direct Overview

Connect directly to Sendexa's SMPP (Short Message Peer-to-Peer) gateway for enterprise-grade SMS messaging. Perfect for high-volume applications requiring low latency and direct protocol control.

---

## Why Choose SMPP Direct?

### Enterprise Performance

- **High Throughput** - Send thousands of messages per second
- **Low Latency** - Sub-second message delivery
- **Direct Connection** - No HTTP overhead
- **Persistent Connection** - Maintain always-on SMPP sessions
- **Binary Protocol** - Efficient data transmission

### Advanced Features

- **Delivery Reports** - Real-time DLRs via SMPP
- **Long Message Support** - Automatic concatenation
- **Unicode Messages** - Full international character support
- **TLV Parameters** - Extended SMPP functionality
- **Connection Pooling** - Multiple concurrent binds

### Built for Africa

- **Local SMPP Servers** - Low latency across Africa
- **Direct Operator Connections** - Premium routing
- **Competitive Pricing** - Enterprise volume discounts
- **24/7 Support** - Dedicated technical support

## SMPP vs HTTP API

| Feature        | SMPP Direct            | HTTP API            |
| -------------- | ---------------------- | ------------------- |
| **Throughput** | 1000+ msgs/sec         | 60 msgs/min         |
| **Latency**    | < 100ms                | 200-500ms           |
| **Connection** | Persistent             | Request/Response    |
| **Protocol**   | Binary                 | JSON/REST           |
| **Complexity** | Higher                 | Lower               |
| **Use Case**   | Enterprise/High-volume | Standard/Low-volume |
| **Setup**      | SMPP Client required   | Simple HTTP calls   |

**When to use SMPP:**

- Sending 10,000+ messages per day
- Need sub-second latency
- Require persistent connections
- Have existing SMPP infrastructure
- Need maximum throughput

**When to use HTTP API:**

- Quick integration needed
- Lower volume messaging
- No SMPP client available
- Simpler setup preferred

## Getting Started

### 1. Create SMPP Account

1. Log in to [dashboard.sendexa.co](https://dashboard.sendexa.co)
2. Navigate to **SMPP â†’ Accounts**
3. Click **Create SMPP Account**
4. Note your credentials:
   - System ID (username)
   - Password
   - Allowed IPs (for security)

### 2. Whitelist Your IP

For security, whitelist your server IPs:

1. Go to **SMPP â†’ IP Whitelist**
2. Add your server IP addresses
3. Click **Save**

### 3. Test Connection

Use telnet to test connectivity:

```bash
telnet smpp.sendexa.co 2775
```

If successful, you'll see a connection established.

### 4. Install SMPP Client

Choose an SMPP client library for your platform:

**Node.js:**

```bash
npm install smpp
```

**Python:**

```bash
pip install smpplib
```

**PHP:**

```bash
composer require onlinecity/php-smpp
```

**Java:**

```bash
# Add to pom.xml
<dependency>
    <groupId>org.jsmpp</groupId>
    <artifactId>jsmpp</artifactId>
    <version>2.3.11</version>
</dependency>
```

### 5. Connect and Send

See [Connect to SMPP](/docs/smpp/connect) for detailed connection examples.

### Message Lifecycle

1. **Bind** - Establish SMPP connection
2. **Submit_SM** - Send message
3. **Submit_SM_Resp** - Receive message ID
4. **Deliver_SM** - Receive delivery report
5. **Unbind** - Close connection (optional)

---

## Supported Operations

### Transmitter (TX) Operations

- `bind_transmitter` - Send messages only
- `submit_sm` - Submit single message
- `submit_multi` - Submit to multiple recipients
- `data_sm` - Send data message
- `unbind` - Close connection

### Receiver (RX) Operations

- `bind_receiver` - Receive DLRs only
- `deliver_sm` - Receive delivery reports
- `unbind` - Close connection

### Transceiver (TRX) Operations

- `bind_transceiver` - Send and receive (recommended)
- All TX and RX operations

**Recommended:** Use `bind_transceiver` for full functionality.

## Delivery Reports (DLR)

### DLR Request

Set `registered_delivery` parameter to request DLR:

```
registered_delivery = 0x01  # Request DLR
```

### DLR Status Codes

| Status  | Code | Description          |
| ------- | ---- | -------------------- |
| DELIVRD | 0    | Delivered to handset |
| EXPIRED | 1    | Message expired      |
| DELETED | 2    | Deleted by operator  |
| UNDELIV | 3    | Undeliverable        |
| ACCEPTD | 4    | Accepted by operator |
| UNKNOWN | 5    | Unknown status       |
| REJECTD | 6    | Rejected by operator |

### DLR Format

## Best Practices

### 1. Use Connection Pooling

```javascript
// Maintain multiple connections
const connectionPool = [];
for (let i = 0; i < 5; i++) {
  const session = await createSMPPConnection();
  connectionPool.push(session);
}

// Round-robin distribution
let currentIndex = 0;
function getNextConnection() {
  const conn = connectionPool[currentIndex];
  currentIndex = (currentIndex + 1) % connectionPool.length;
  return conn;
}
```

### 2. Implement Enquire Link

Keep connections alive with periodic pings:

```javascript
setInterval(() => {
  session.enquire_link({}, (pdu) => {
    if (pdu.command_status === 0) {
      console.log('Connection alive');
    }
  });
}, 30000); // Every 30 seconds
```

### 3. Handle Reconnection

```javascript
function connect() {
  session.connect();

  session.on('close', () => {
    console.log('Connection closed, reconnecting...');
    setTimeout(connect, 5000);
  });

  session.on('error', (err) => {
    console.error('SMPP error:', err);
  });
}
```

### 4. Queue Messages

Implement message queue for reliability:

```javascript
const messageQueue = [];

function queueMessage(message) {
  messageQueue.push(message);
  processQueue();
}

async function processQueue() {
  while (messageQueue.length > 0) {
    const message = messageQueue[0];
    try {
      await sendSMS(message);
      messageQueue.shift(); // Remove on success
    } catch (error) {
      console.error('Send failed, retrying...', error);
      await sleep(5000);
    }
  }
}
```

### 5. Monitor Connection Health

```javascript
let messagesPerMinute = 0;
let lastCheck = Date.now();

setInterval(() => {
  const now = Date.now();
  const elapsed = (now - lastCheck) / 1000 / 60;
```

## Troubleshooting

### Connection Refused

**Problem:** Cannot connect to SMPP server

**Solutions:**

1. Check if IP is whitelisted
2. Verify correct host and port
3. Check firewall rules
4. Test with telnet

### Bind Failed

**Problem:** Bind request rejected

**Solutions:**

1. Verify system_id and password
2. Check if account is active
3. Ensure not exceeding concurrent bind limit
4. Review error code in bind_resp

### Throttling Errors

**Problem:** Receiving ESME_RTHROTTLED

**Solutions:**

1. Reduce message rate
2. Implement exponential backoff
3. Use connection pooling
4. Upgrade to higher tier

### Messages Not Delivered

**Problem:** Submit successful but no delivery

**Solutions:**

1. Check DLR for error details
2. Verify recipient number format
3. Ensure sufficient balance
4. Check network availability

## Migration from HTTP API

### Step-by-Step Migration

1. **Setup SMPP account** in dashboard
2. **Install SMPP client** library
3. **Test connection** in sandbox
4. **Implement basic send** functionality
5. **Add DLR handling**
6. **Run parallel** (SMPP + HTTP) for 1 week
7. **Monitor metrics** for comparison
8. **Switch to SMPP** when stable
9. **Decommission HTTP** integration

**Before (HTTP):**

```javascript
await axios.post('https://api.sendexa.co/v1/sms/send', {
```
