## Key Features

### Quick Send
Send notifications to individual devices or users instantly:
- Transactional alerts
- Order updates
- Breaking news
- Personal messages

**Endpoint:** `POST /v1/push/send`

```json
{
  "device_token": "fL9x8K3mR2...",
  "title": "Order Shipped",
  "body": "Your order #12345 is on the way!",
  "data": {
    "orderId": "12345",
    "action": "track_order"
  }
}
```

### Bulk Send
Send notifications to multiple users with one API call:
- Marketing campaigns
- App updates
- Event announcements
- Mass alerts

**Endpoint:** `POST /v1/push/send` (with array of tokens)

```json
{
  "device_tokens": ["token1", "token2", "token3"],
  "title": "Flash Sale!",
  "body": "50% off all items. Limited time only!",
  "image": "https://cdn.yourapp.com/sale-banner.jpg"
}
```

### Topic Broadcasting
Send to all subscribers of a topic:
- News categories
- Sports updates
- Location-based alerts
- Interest groups

**Endpoint:** `POST /v1/push/send/topic`

```json
{
  "topic": "sports_football",
  "title": "Goal!",
  "body": "Manchester United scores in the 90th minute!"
}
```

### Rich Notifications
Enhance notifications with media and actions:
- Images and videos
- Action buttons
- Deep links
- Custom sounds

## Getting Started

### 1. Create an Account
Sign up at [app.sendexa.co](https://app.sendexa.co) and verify your email.

### 2. Generate API Token
1. Navigate to **Settings ‚Üí API Tokens**
2. Click **Create New Token**
3. Copy your token securely
4. Use it in the `Authorization` header

```http
Authorization: Bearer sk_live_your_token_here
```

### 3. Register Your App
1. Go to **Settings ‚Üí Push Apps**
2. Click **Add App**
3. Provide:
   - App name
   - Platform (iOS/Android/Web)
   - Upload credentials (APNs/FCM)
4. Get your App ID

### 4. Integrate SDK

**iOS (Swift)**
```swift
import SendexaPush

SendexaPush.initialize(appId: "your_app_id")
SendexaPush.registerForPushNotifications()
```

**Android (Kotlin)**
```kotlin
import co.sendexa.push.SendexaPush

SendexaPush.initialize(this, "your_app_id")
SendexaPush.registerDevice()
```

**React Native**
```javascript
import SendexaPush from '@sendexa/react-native-push';

await SendexaPush.initialize('your_app_id');
await SendexaPush.requestPermissions();
```

**Web (JavaScript)**
```javascript
import { SendexaPush } from '@sendexa/web-push';

const push = new SendexaPush('your_app_id');
await push.requestPermission();
```

### 5. Send Your First Notification

```bash
curl -X POST 'https://api.sendexa.co/v1/push/send' \
  -H 'Authorization: Bearer sk_live_your_token' \
  -H 'Content-Type: application/json' \
  -d '{
    "device_token": "fL9x8K3mR2nQ5pW7...",
    "title": "Welcome to Sendexa!",
    "body": "Thanks for joining us. Start exploring now!",
    "data": {
      "screen": "home"
    }
  }'
```

## Authentication

Sendexa uses **Bearer Token authentication**. Include your API token in the `Authorization` header of every request:

```http
Authorization: Bearer YOUR_API_TOKEN
```

**Security Best Practices:**
- Never expose API tokens in client-side code
- Use environment variables to store tokens
- Rotate tokens regularly
- Use different tokens for different environments
- Revoke unused tokens immediately

## API Endpoints Summary

| Endpoint | Method | Purpose | Rate Limit |
|----------|--------|---------|------------|
| `/v1/push/send` | POST | Send to device(s) | 100/min |
| `/v1/push/send/topic` | POST | Send to topic subscribers | 60/min |
| `/v1/push/devices` | GET | List registered devices | 60/min |
| `/v1/push/devices/:id` | GET | Get device details | 100/min |
| `/v1/push/devices/:id` | DELETE | Remove device | 60/min |
| `/v1/push/topics/:topic/subscribe` | POST | Subscribe to topic | 100/min |
| `/v1/push/topics/:topic/unsubscribe` | POST | Unsubscribe from topic | 100/min |
| `/v1/push/analytics` | GET | Get delivery analytics | 30/min |

## Best Practices

### 1. Notification Content
- **Be Clear:** Use concise, actionable language
- **Be Relevant:** Send personalized, timely content
- **Be Valuable:** Provide real value to users
- **Be Respectful:** Don't spam or annoy users

### 2. Timing
- Respect user time zones
- Avoid late-night notifications
- Consider user behavior patterns
- Use quiet hours (10 PM - 8 AM)

### 3. Frequency
- Don't overwhelm users
- Set frequency caps
- Allow users to control preferences
- Monitor opt-out rates

### 4. Targeting
- Segment your audience
- Personalize content
- Use A/B testing
- Track engagement metrics

### 5. Permission Handling
- Request permission at the right time
- Explain the value clearly
- Respect user choices
- Make opt-out easy

## Migration Guide

### From Firebase Cloud Messaging (FCM)

Migrating from FCM to Sendexa Push:

**Key Differences:**
- Unified API for iOS and Android
- Built-in analytics
- Topic management
- Better delivery rates in Africa

**Migration Steps:**
1. Create Sendexa app
2. Upload FCM credentials
3. Update SDK initialization
4. Update token registration
5. Update notification sending code
6. Test thoroughly

**Code Changes:**

**Before (FCM):**
```javascript
// FCM
const messaging = firebase.messaging();
const token = await messaging.getToken();

// Send via FCM Admin SDK
await admin.messaging().send({
  token: token,
  notification: { title, body }
});
```

**After (Sendexa):**
```javascript
// Sendexa
import SendexaPush from '@sendexa/sdk';

const token = await SendexaPush.getDeviceToken();

// Send via Sendexa API
await fetch('https://api.sendexa.co/v1/push/send', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer sk_live_your_token',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({ device_token: token, title, body })
});
```
