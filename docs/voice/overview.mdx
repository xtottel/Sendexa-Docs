## Key Features

### Make a Call

Place outbound calls to any phone number:

- Person-to-person calls
- Automated notifications
- Appointment reminders
- Delivery confirmations

**Endpoint:** `POST /v1/voice/call`

```json
{
  "to": "233501234567",
  "from": "233302123456",
  "url": "https://yourapp.com/voice.xml"
}
```

### Call Broadcasting

Send voice messages to multiple recipients:

- Emergency alerts
- Political campaigns
- Event announcements
- Marketing campaigns

**Endpoint:** `POST /v1/voice/broadcast`

```json
{
  "to": ["233501234567", "233509876543"],
  "from": "233302123456",
  "message": "This is an important announcement",
  "voice": "female"
}
```

### Text-to-Speech (TTS)

Convert text to natural-sounding speech:

- Multi-language support
- Multiple voice options
- Custom pronunciation
- SSML support

**Endpoint:** `POST /v1/voice/tts`

```json
{
  "text": "Hello, your order has been delivered",
  "language": "en-US",
  "voice": "female"
}
```

### Interactive Voice Response (IVR)

Build interactive voice menus:

- Customer support systems
- Automated surveys
- Payment systems
- Appointment booking

**Using VoiceXML:**

```xml
<Response>
  <Gather numDigits="1" action="/handle-choice">
    <Say>Press 1 for Sales, 2 for Support</Say>
  </Gather>
</Response>
```

## Getting Started

### 1. Create an Account

Sign up at [app.sendexa.co](https://app.sendexa.co) and verify your email.

### 2. Generate API Token

1. Navigate to **Settings â†’ API Tokens**
2. Click **Create New Token**
3. Copy your token securely
4. Use it in the `Authorization` header

```http
Authorization: Bearer sk_live_your_token_here
```

### 3. Get a Phone Number

1. Go to **Voice â†’ Phone Numbers**
2. Click **Buy Number**
3. Select country and number type
4. Configure voice URL endpoint
5. Start making calls

### 4. Make Your First Call

```bash
curl -X POST 'https://api.sendexa.co/v1/voice/call' \
  -H 'Authorization: Bearer sk_live_your_token' \
  -H 'Content-Type: application/json' \
  -d '{
    "to": "233501234567",
    "from": "233302123456",
    "url": "https://yourapp.com/voice.xml"
  }'
```

## Authentication

Sendexa uses **Bearer Token authentication**. Include your API token in the `Authorization` header of every request:

```http
Authorization: Bearer YOUR_API_TOKEN
```

**Security Best Practices:**

- Never expose API tokens in client-side code
- Use environment variables to store tokens
- Rotate tokens regularly
- Use different tokens for different environments
- Revoke unused tokens immediately

## VoiceXML (Voice Markup)

Sendexa uses VoiceXML to control call flow. When a call is answered, Sendexa fetches VoiceXML from your specified URL.

### Basic VoiceXML Structure

```xml
<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Say voice="female">Hello, welcome to our service</Say>
  <Play>https://yourapp.com/audio/welcome.mp3</Play>
  <Gather numDigits="1" action="/handle-input">
    <Say>Press 1 for Sales, 2 for Support</Say>
  </Gather>
</Response>
```

### VoiceXML Verbs

| Verb         | Purpose                   | Example                         |
| ------------ | ------------------------- | ------------------------------- |
| `<Say>`      | Speak text using TTS      | `<Say>Hello World</Say>`        |
| `<Play>`     | Play audio file           | `<Play>audio.mp3</Play>`        |
| `<Gather>`   | Collect DTMF input        | `<Gather numDigits="4">`        |
| `<Record>`   | Record caller's voice     | `<Record maxLength="60">`       |
| `<Dial>`     | Connect to another number | `<Dial>233501234567</Dial>`     |
| `<Hangup>`   | End the call              | `<Hangup/>`                     |
| `<Redirect>` | Redirect to new URL       | `<Redirect>/new-url</Redirect>` |
| `<Pause>`    | Add silence               | `<Pause length="2"/>`           |

## Common Use Cases

### 1. Order Confirmation Calls

```javascript
// Make automated order confirmation call
await makeCall({
  to: '+233501234567',
  from: '233302123456',
  url: 'https://yourapp.com/order-confirmation.xml',
});

// order-confirmation.xml
<Response>
  <Say>Hello, this is a confirmation for order number 12345.</Say>
  <Say>Your order will be delivered tomorrow between 10 AM and 2 PM.</Say>
  <Say>Thank you for shopping with us!</Say>
</Response>;
```

### 2. Appointment Reminders

```javascript
// Send appointment reminder
await makeCall({
  to: '+233501234567',
  from: '233302123456',
  url: 'https://yourapp.com/appointment-reminder.xml?name=John&time=2PM',
});

// appointment-reminder.xml
<Response>
  <Say>
    Hello {{ name }}, this is a reminder about your appointment tomorrow at{' '}
    {{ time }}.
  </Say>
  <Gather numDigits="1" action="/confirm-appointment">
    <Say>Press 1 to confirm, or 2 to reschedule.</Say>
  </Gather>
</Response>;
```

### 3. Customer Support IVR

```xml
<Response>
  <Say>Welcome to Customer Support</Say>
  <Gather numDigits="1" action="/handle-menu">
    <Say>Press 1 for Account Information</Say>
    <Say>Press 2 for Technical Support</Say>
    <Say>Press 3 to speak with an agent</Say>
    <Say>Press 0 to repeat this menu</Say>
  </Gather>
</Response>
```

### 4. Emergency Broadcast

```javascript
// Broadcast emergency alert
await broadcastCall({
  to: emergencyContactNumbers,
  from: '233302123456',
  message:
    'Emergency alert: Building evacuation in progress. Please exit immediately.',
  priority: 'high',
});
```

## Call Events & Webhooks

Sendexa sends webhook notifications for call events:

### Events

| Event                 | Description             |
| --------------------- | ----------------------- |
| `call.initiated`      | Call initiated          |
| `call.ringing`        | Phone is ringing        |
| `call.answered`       | Call was answered       |
| `call.completed`      | Call ended successfully |
| `call.failed`         | Call failed             |
| `call.busy`           | Number was busy         |
| `call.no-answer`      | No one answered         |
| `recording.completed` | Recording ready         |

### Webhook Payload

```json
{
  "event": "call.answered",
  "call_id": "call_abc123",
  "from": "233302123456",
  "to": "233501234567",
  "status": "in-progress",
  "duration": 0,
  "timestamp": "2025-03-12T14:20:00Z"
}
```

## Best Practices

### 1. Voice Quality

- Use clear, professional voice messages
- Test TTS with different voices
- Keep messages concise
- Use appropriate pacing

### 2. User Experience

- Provide clear instructions
- Offer option to repeat menu
- Allow users to go back
- Keep IVR menus shallow (max 3 levels)

### 3. Timing

- Respect calling hours (9 AM - 8 PM)
- Consider time zones
- Avoid holidays and weekends for non-urgent calls
- Set retry limits

### 4. Call Flow

- Start with identification
- Provide clear menu options
- Confirm user input
- Always offer escape option

### 5. Compliance

- Get consent before automated calls
- Provide opt-out mechanism
- Follow local calling regulations
- Maintain Do Not Call list

## Call Recording

### Enable Recording

```xml
<Response>
  <Say>This call may be recorded for quality purposes</Say>
  <Record
    maxLength="300"
    playBeep="true"
    action="/handle-recording"
  />
</Response>
```

### Retrieve Recording

```bash
curl -X GET 'https://api.sendexa.co/v1/voice/calls/call_abc123/recording' \
  -H 'Authorization: Bearer sk_live_your_token'
```

### Response

```json
{
  "success": true,
  "recording": {
    "url": "https://recordings.sendexa.co/call_abc123.mp3",
    "duration": 45,
    "size": 720000,
    "format": "mp3",
    "created_at": "2025-03-12T14:25:00Z"
  }
}
```
