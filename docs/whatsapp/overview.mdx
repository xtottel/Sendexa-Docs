## Key Features

### Send Messages
Send text, media, and interactive messages instantly.

```json
{
  "to": "233501234567",
  "type": "text",
  "text": {
    "body": "Hello from Sendexa!"
  }
}
```

### Broadcasts
Send messages to multiple contacts at once.

```json
{
  "name": "March Promotion",
  "contacts": ["233501234567", "233509876543"],
  "message": {
    "type": "template",
    "template": {
      "name": "promotion_alert",
      "language": "en"
    }
  }
}
```

### Contact Management
Organize contacts with tags and custom fields.

```json
{
  "phone": "233501234567",
  "name": "John Doe",
  "email": "john@example.com",
  "tags": ["customer", "vip"]
}
```

### Message Templates
Create and manage WhatsApp approved templates.

```json
{
  "name": "order_confirmation",
  "category": "TRANSACTIONAL",
  "language": "en",
  "components": [
    {
      "type": "BODY",
      "text": "Your order {{1}} has been confirmed!"
    }
  ]
}
```

## Getting Started

### 1. Create an Account
Sign up at [app.sendexa.co](https://app.sendexa.co) and verify your email.

### 2. Connect WhatsApp Business
1. Navigate to **WhatsApp â†’ Settings**
2. Click **Connect WhatsApp Business**
3. Follow Meta verification process
4. Complete business profile

### 3. Generate API Token
1. Go to **Settings â†’ API Tokens**
2. Click **Create New Token**
3. Select **WhatsApp** scope
4. Copy your token securely

```http
Authorization: Bearer sk_live_your_token_here
```

### 4. Create Your First Template
Before sending messages, create an approved template:

1. Go to **WhatsApp â†’ Templates**
2. Click **Create Template**
3. Fill template details
4. Submit for Meta approval (24-48 hours)

### 5. Send Your First Message

```bash
curl -X POST 'https://api.sendexa.co/v1/whatsapp/send' \
  -H 'Authorization: Bearer sk_live_your_token' \
  -H 'Content-Type: application/json' \
  -d '{
    "to": "233501234567",
    "type": "template",
    "template": {
      "name": "hello_world",
      "language": "en"
    }
  }'
```

## Authentication

Use **Bearer Token authentication** in the `Authorization` header:

```http
Authorization: Bearer YOUR_API_TOKEN
```

**Security Best Practices:**
- Never expose API tokens in client-side code
- Use environment variables
- Rotate tokens regularly
- Use different tokens for different environments
- Revoke unused tokens immediately

## Message Pricing

WhatsApp uses conversation-based pricing. A 24-hour conversation window opens when:
- You send a template message
- User sends you a message

### Ghana (GHS)
| Conversation Type | Price |
|-------------------|-------|
| User-initiated | 0.25 GHS |
| Business-initiated (Marketing) | 0.45 GHS |
| Business-initiated (Utility) | 0.35 GHS |
| Business-initiated (Authentication) | 0.30 GHS |

### Nigeria (NGN)
| Conversation Type | Price |
|-------------------|-------|
| User-initiated | 15 NGN |
| Business-initiated (Marketing) | 30 NGN |
| Business-initiated (Utility) | 20 NGN |
| Business-initiated (Authentication) | 18 NGN |

### Kenya (KES)
| Conversation Type | Price |
|-------------------|-------|
| User-initiated | 3.50 KES |
| Business-initiated (Marketing) | 6.00 KES |
| Business-initiated (Utility) | 4.50 KES |
| Business-initiated (Authentication) | 4.00 KES |

**Notes:**
- Prices are per 24-hour conversation window
- Unlimited messages within the window
- Free session messages (replies within 24h of user message)

## Message Types

### Text Messages
Simple text with optional preview URL.

```json
{
  "type": "text",
  "text": {
    "body": "Hello! Check our website: https://example.com",
    "preview_url": true
  }
}
```

### Media Messages
Send images, videos, documents, or audio.

```json
{
  "type": "image",
  "image": {
    "url": "https://example.com/image.jpg",
    "caption": "Check out our new product!"
  }
}
```

### Interactive Messages
Buttons and list selections.

```json
{
  "type": "interactive",
  "interactive": {
    "type": "button",
    "body": {
      "text": "Choose an option:"
    },
    "action": {
      "buttons": [
        {
          "type": "reply",
          "reply": {
            "id": "yes",
            "title": "Yes"
          }
        },
        {
          "type": "reply",
          "reply": {
            "id": "no",
            "title": "No"
          }
        }
      ]
    }
  }
}
```

### Template Messages
Pre-approved templates for notifications.

```json
{
  "type": "template",
  "template": {
    "name": "order_update",
    "language": "en",
    "components": [
      {
        "type": "body",
        "parameters": [
          {
            "type": "text",
            "text": "ORD-12345"
          }
        ]
      }
    ]
  }
}
```

## Common Use Cases

### 1. Order Notifications
```javascript
await sendWhatsApp({
  to: "+233501234567",
  type: "template",
  template: {
    name: "order_confirmation",
    language: "en",
    components: [{
      type: "body",
      parameters: [
        { type: "text", text: "ORD-12345" },
        { type: "text", text: "GHS 150.00" }
      ]
    }]
  }
});
```

### 2. Customer Support
```javascript
// Receive user message via webhook
// Reply with session message (free within 24h)
await sendWhatsApp({
  to: "+233501234567",
  type: "text",
  text: {
    body: "Thanks for reaching out! How can I help you today?"
  }
});
```

### 3. Marketing Campaigns
```javascript
// Send to segmented audience
await createBroadcast({
  name: "Flash Sale Alert",
  contacts: vipCustomers,
  message: {
    type: "template",
    template: {
      name: "flash_sale",
      language: "en"
    }
  }
});
```

### 4. Appointment Reminders
```javascript
await sendWhatsApp({
  to: "+233501234567",
  type: "template",
  template: {
    name: "appointment_reminder",
    language: "en",
    components: [{
      type: "body",
      parameters: [
        { type: "text", text: "Dr. Smith" },
        { type: "text", text: "March 15, 2025 at 2:00 PM" }
      ]
    }]
  }
});
```

## Best Practices

### 1. Message Templates
- Get templates approved before campaigns
- Use appropriate template categories
- Keep templates simple and clear
- Test templates before bulk sending

### 2. Conversation Management
- Reply within 24-hour windows when possible
- Use session messages for free replies
- Track conversation costs
- Optimize template usage

### 3. Contact Management
- Keep contact lists organized
- Use tags for segmentation
- Update contact info regularly
- Handle opt-outs properly

### 4. Media Handling
- Optimize image/video sizes
- Use supported formats
- Host media on reliable CDN
- Include captions for context

### 5. Compliance
- Get user opt-in before messaging
- Provide opt-out mechanism
- Follow WhatsApp Business Policy
- Respect user preferences

## WhatsApp Business Requirements

### Verification Required
- **Business Name** - Official registered name
- **Business Website** - Active website URL
- **Business Description** - What your business does
- **Business Category** - Industry category
- **Profile Photo** - Business logo

### Display Name
Your business name appears as the sender. Must be:
- Actual business name
- Approved by Meta
- Not misleading
- Not impersonating others
