# Overview

Welcome to the official Sendexa Developer Documentation. This guide provides a complete overview of our communication platform, covering SMS, OTP, Email, Voice (future), and all tools designed to help businesses integrate Sendexa seamlessly into their workflows.

## What is Sendexa?
Sendexa is a modern communication platform built for businesses in Ghana and across Africa. We provide reliable, fast, and scalable APIs for:

- **SMS Messaging** (Transactional & Marketing)
- **OTP & Verification** (SMS & Email OTP)
- **Email Communication** (Transactional emails)
- **Voice Services** (upcoming)
- **Webhooks & Event-driven integrations**

Our goal is to empower developers with simple yet powerful APIs and dashboards that help businesses communicate at scale.

## Why Sendexa?
- **Built for Africa-first scalability**
- **Multiple provider routing** (Kairos, Telecel, etc.)
- **Simple authentication** with Token-based authorization
- **Fast delivery & intelligent routing**
- **Transparent pricing**
- **Developer-friendly SDKs & examples**
- **24/7 system monitoring & uptime guarantees** (post-launch)

## Core API Categories
- **SMS API** – Send single or bulk messages with advanced features
- **OTP API** – Request, verify, and resend OTPs for any application
- **Email API** – Send templated transactional emails
- **Webhook API** – Subscribe to delivery reports and events

## Base URLs
Different services run on dedicated subdomains for performance and scalability:

- **SMS API** → `https://smsc.sendexa.co`
- **OTP API** → `https://2fa.sendexa.co`
- **Email API** → `https://email.sendexa.co` (coming soon)

## Authentication
Sendexa v2 uses **Token-based Authorization** instead of legacy API keys.

Example:
```http
Authorization: Bearer YOUR_TOKEN_HERE
```
