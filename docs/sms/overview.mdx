## Key Features

### Quick Send
Send SMS to a single recipient instantly. Perfect for:
- OTP verification codes
- Transactional alerts
- Appointment reminders
- Order confirmations

**Endpoint:** `POST /v1/sms/send`

```json
{
  "to": "233501234567",
  "from": "YourBrand",
  "content": "Your OTP is 123456"
}
```

### Bulk Send
Send the same message to multiple recipients with one API call:
- Marketing campaigns
- Announcements
- Mass notifications
- Event reminders

**Endpoint:** `POST /v1/sms/send` (with array of numbers)

```json
{
  "to": ["233501234567", "233509876543"],
  "from": "YourBrand",
  "content": "Flash Sale! 50% off today only"
}
```

### Delivery Reports
Track message delivery in real-time:
- Check delivery status
- Get delivery timestamps
- Monitor failed messages
- Receive webhook notifications

**Endpoint:** `GET /v1/sms/status/:messageId`

### Smart Resend
Automatically retry failed messages:
- Up to 3 retry attempts
- Exponential backoff
- Different provider routing
- Cost-effective retries

**Endpoint:** `POST /v1/sms/resend/:messageId`

## Getting Started

### 1. Create an Account
Sign up at [app.sendexa.co](https://app.sendexa.co) and verify your email.

### 2. Generate API Token
1. Navigate to **Settings â†’ API Tokens**
2. Click **Create New Token**
3. Copy your token securely
4. Use it in the `Authorization` header

```http
Authorization: Bearer sk_live_your_token_here
```

### 3. Register Sender ID
Before sending SMS:
1. Go to **Settings â†’ Sender IDs**
2. Submit your desired Sender ID
3. Await approval (usually within 24 hours)
4. Use approved Sender ID in your requests

### 4. Send Your First SMS

```bash
curl -X POST 'https://api.sendexa.co/v1/sms/send' \
  -H 'Authorization: Bearer sk_live_your_token' \
  -H 'Content-Type: application/json' \
  -d '{
    "to": "233501234567",
    "from": "YourBrand",
    "content": "Hello from Sendexa!"
  }'
```

## Authentication

Sendexa uses **Bearer Token authentication**. Include your API token in the `Authorization` header of every request:

```http
Authorization: Bearer YOUR_API_TOKEN
```

**Security Best Practices:**
- Never expose API tokens in client-side code
- Use environment variables to store tokens
- Rotate tokens regularly
- Use different tokens for different environments
- Revoke unused tokens immediately

## Common Use Cases

### 1. Authentication & Verification
```javascript
// Send OTP for login
await sendSMS({
  to: "+233501234567",
  from: "YourApp",
  content: "Your verification code is 123456. Valid for 10 minutes."
});
```

### 2. Transactional Notifications
```javascript
// Order confirmation
await sendSMS({
  to: "+233501234567",
  from: "YourShop",
  content: "Order #12345 confirmed! Estimated delivery: 2-3 days."
});
```

### 3. Marketing Campaigns
```javascript
// Bulk promotional SMS
await sendBulkSMS({
```

## Best Practices

### 1. Message Content
- Keep messages clear and concise
- Include sender identification
- Provide opt-out instructions for marketing
- Use proper grammar and spelling
- Avoid ALL CAPS (looks like spam)

### 2. Phone Number Formatting
Always use international format:
- âœ… **Correct:** `+233501234567`
- âŒ **Wrong:** `0501234567`
- âŒ **Wrong:** `233501234567` (missing +)

### 3. Sender ID Guidelines
- 3-11 characters max
- Alphanumeric only (no special characters)
- Use your brand name for recognition
- Register before using
- Don't use generic IDs like "INFO" or "ALERT"

### 4. Rate Limiting
- Respect rate limits (60 requests/min for SMS)
- Use bulk endpoints for multiple recipients
- Implement exponential backoff for retries
- Cache frequently used data

### 5. Error Handling
- Always check response status
- Log errors with message IDs
- Implement retry logic for transient errors
- Monitor failed messages dashboard

### 6. Webhooks
- Use webhooks instead of polling
- Verify webhook signatures
- Respond with 200 OK quickly
- Process webhook data asynchronously

## Migration Guide

### From v0 to v1

If you're upgrading from our legacy v0 API:

**What Changed:**
- New authentication (Bearer tokens vs API keys)
- Simplified request structure
- Enhanced error messages
- Real-time webhooks

**Migration Steps:**
1. Generate new API token from dashboard
2. Update base URL to `v1`
3. Change authentication header
4. Update request body structure
5. Update webhook endpoint

**Example Changes:**

**v0 (Old):**
```json
{
  "api_key": "your_key",
  "recipient": "233501234567",
  "sender": "Brand",
  "message": "Hello"
}
```

**v1 (New):**
```json
{
  "to": "233501234567",
  "from": "Brand",
  "content": "Hello"
}
```
