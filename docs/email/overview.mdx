## Key Features

### Send Email

Send transactional and marketing emails:

- Single or bulk sending
- HTML and plain text support
- Custom from name and email
- CC and BCC recipients
- Reply-to configuration

**Endpoint:** `POST /v1/email/send`

```json
{
  "to": "user@example.com",
  "from": "noreply@yourdomain.com",
  "subject": "Welcome to YourApp",
  "html": "<h1>Welcome!</h1><p>Thanks for signing up.</p>",
  "text": "Welcome! Thanks for signing up."
}
```

### Email Templates

Create and manage reusable templates:

- HTML template editor
- Variable substitution
- Template versioning
- Preview before sending

**Endpoint:** `POST /v1/email/send` with `templateId`

```json
{
  "to": "user@example.com",
  "templateId": "welcome_email",
  "variables": {
    "name": "John Doe",
    "verificationLink": "https://app.com/verify/123"
  }
}
```

### Attachments

Send files with your emails:

- Multiple file support
- Base64 encoding
- Common file types (PDF, images, documents)
- Size limits: 10MB per email

**Example:**

```json
{
  "to": "user@example.com",
  "subject": "Your Invoice",
  "attachments": [
    {
      "filename": "invoice.pdf",
      "content": "base64_encoded_content",
      "type": "application/pdf"
    }
  ]
}
```

### Webhooks

Real-time email event notifications:

- Email delivered
- Email opened
- Link clicked
- Bounce notifications
- Spam complaints

## Getting Started

### 1. Create an Account

Sign up at [dashboard.sendexa.co](https://dashboard.sendexa.co) and verify your email.

### 2. Verify Your Domain

1. Go to **Settings â†’ Domains**
2. Add your sending domain
3. Add DNS records (SPF, DKIM, DMARC)
4. Wait for verification (usually 24 hours)

### 3. Generate API Token

1. Navigate to **Settings â†’ API Tokens**
2. Click **Create New Token**
3. Copy your token securely
4. Use it in the `Authorization` header

```http
Authorization: Bearer YOUR_API_TOKEN
```

### 4. Send Your First Email

```bash
curl -X POST 'https://api.sendexa.co/v1/email/send' \
  -H 'Authorization: Bearer YOUR_API_TOKEN' \
  -H 'Content-Type: application/json' \
  -d '{
    "to": "user@example.com",
    "from": "hello@yourdomain.com",
    "fromName": "YourApp",
    "subject": "Welcome!",
    "html": "<h1>Welcome to YourApp!</h1>"
  }'
```

## Common Use Cases

### 1. Welcome Email

```javascript
await sendEmail({
  to: 'newuser@example.com',
  from: 'welcome@yourapp.com',
  fromName: 'YourApp Team',
  subject: 'Welcome to YourApp!',
  templateId: 'welcome_email',
  variables: {
    name: 'John Doe',
    verificationLink: 'https://app.com/verify/abc123',
  },
});
```

### 2. Order Confirmation

```javascript
await sendEmail({
  to: 'customer@example.com',
  from: 'orders@yourstore.com',
  subject: 'Order #12345 Confirmed',
  templateId: 'order_confirmation',
  variables: {
    orderNumber: '12345',
    items: orderItems,
    total: '500.00',
    currency: 'GHS',
  },
  attachments: [
    {
      filename: 'invoice.pdf',
      content: invoicePdfBase64,
      type: 'application/pdf',
    },
  ],
});
```

### 3. Password Reset

```javascript
await sendEmail({
  to: 'user@example.com',
  from: 'security@yourapp.com',
  subject: 'Reset Your Password',
  html: `
    <h2>Password Reset Request</h2>
    <p>Click the link below to reset your password:</p>
    <a href="${resetLink}">Reset Password</a>
    <p>This link expires in 1 hour.</p>
  `,
});
```

### 4. Weekly Newsletter

```javascript
const subscribers = await getNewsletterSubscribers();

await sendBatchEmail({
  from: 'newsletter@yourapp.com',
  subject: 'This Week in Tech',
  templateId: 'newsletter_weekly',
  recipients: subscribers.map((sub) => ({
    to: sub.email,
    variables: {
      name: sub.name,
      unsubscribeLink: `https://app.com/unsubscribe/${sub.id}`,
    },
  })),
});
```

### 5. Invoice Delivery

```javascript
await sendEmail({
  to: 'client@company.com',
  from: 'billing@yourcompany.com',
  subject: `Invoice #${invoiceNumber}`,
  html: `
    <h2>Invoice #${invoiceNumber}</h2>
    <p>Thank you for your business. Please find your invoice attached.</p>
  `,
  attachments: [
    {
      filename: `invoice-${invoiceNumber}.pdf`,
      content: pdfBase64,
      type: 'application/pdf',
    },
  ],
});
```

### Status Descriptions

| Status         | Description                           |
| -------------- | ------------------------------------- |
| **queued**     | Email is in send queue                |
| **sent**       | Email sent to recipient's mail server |
| **delivered**  | Email accepted by recipient's server  |
| **opened**     | Recipient opened the email            |
| **clicked**    | Recipient clicked a link              |
| **bounced**    | Email bounced (hard or soft)          |
| **complained** | Marked as spam by recipient           |

## Domain Verification

### Why Verify Your Domain?

- **Higher deliverability** - Emails less likely to be marked as spam
- **Custom sender** - Send from your own domain
- **Professional appearance** - Build trust with recipients
- **Better reputation** - Improve sender reputation

### DNS Records Required

**SPF Record:**

```
v=spf1 include:spf.sendexa.co ~all
```

**DKIM Record:**

```
sendexa._domainkey.yourdomain.com
```

**DMARC Record:**

```
v=DMARC1; p=none; rua=mailto:dmarc@yourdomain.com
```

### Verification Steps

1. Add your domain in dashboard
2. Copy provided DNS records
3. Add records to your DNS provider
4. Wait for propagation (up to 48 hours)
5. Click "Verify" in dashboard

## Best Practices

### Deliverability

1. **Verify your domain** - Essential for good deliverability
2. **Warm up gradually** - Start with low volumes
3. **Clean your list** - Remove invalid addresses
4. **Monitor bounces** - Act on hard bounces immediately
5. **Avoid spam triggers** - Test with spam checkers

### Content

1. **Clear subject lines** - Descriptive and concise
2. **Responsive design** - Mobile-friendly templates
3. **Plain text alternative** - Always provide text version
4. **Unsubscribe link** - Required for marketing emails
5. **Personalization** - Use recipient's name

### Security

1. **Use HTTPS links** - Secure all links in emails
2. **No sensitive data** - Don't send passwords in plain text
3. **Token authentication** - Secure API access
4. **Rate limiting** - Prevent abuse
5. **Monitor activity** - Track unusual patterns

### Performance

1. **Use templates** - Faster and consistent
2. **Batch sending** - For multiple recipients
3. **Optimize images** - Compress before sending
4. **Async processing** - Don't block user flows
5. **Cache results** - Reduce API calls
